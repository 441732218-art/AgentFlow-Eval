## 面向场景与需求

随着大语言模型在客服、办公自动化、知识问答等企业场景中的深入应用，业务系统越来越依赖具备多步骤推理与工具调用能力的 Agent（智能体）。传统人工抽检成本高、覆盖面窄，仅对比最终自然语言答案又难以发现“用错工具、多调工具、中间推理偏离”等问题，导致 Agent 质量回归难以工程化落地。AgentFlow智能体评测管理平台软件面向企业 AI 应用开发者、算法评测人员与质检人员，提供可重复、可追溯、可评分的自动化评测能力，将业务预期固化为测试用例（TestSuite），通过异步或轻量调度执行、完整执行轨迹（Trace）记录与多维指标分（MetricScore）定位问题，并支持对比实验、权限审计与可扩展插件，形成可运行的 Web 工作台形态，支持本地单机与容器化部署，不强制依赖公网即可完整运行。

## 核心功能模块拆解

本软件已实现端到端评测闭环，主要功能包括：

1. **评测任务（Task）管理**：创建、分页查询、状态筛选、执行、取消、归档与删除，任务级 Agent 配置（模型、温度、最大迭代、Runner 类型等）。  
2. **测试用例（TestSuite）管理**：任务下批量维护用例，CSV/JSON 导入，覆盖用户输入、期望输出与期望工具列表。  
3. **Agent 执行与工具沙箱**：OpenAI 兼容 ReAct 多轮执行；HttpAgentRunner 对接外部 Agent HTTP 服务；内置沙箱工具与超时控制。  
4. **执行轨迹（Trace）与可视化**：思考/动作/观察、Token 与耗时结构化落库；前端 DAG、步骤日志与评分卡片联动。  
5. **混合评分与报告**：规则指标 + LLM-as-Judge + 人工复核；报告聚合展示与导出。  
6. **对比实验与 A/B**：多变体 Experiment 对比；粘性分流与统计检验的 A/B 框架。  
7. **企业级横切能力**：API Key 鉴权、RBAC 五角色、租户隔离、审计日志、限流、安全响应头、健康探针与 Prometheus 指标。  
8. **插件与计费扩展（可选）**：插件市场与生命周期钩子；SaaS 配额/订阅骨架（可 mock）。  
9. **双部署剖面**：Lite（零 Redis/Celery Worker）与 Private Docker 全栈共用核心业务路径。

上述功能均已具备可点击、可运行的完整界面与接口形态。

## 技术架构与特点

系统采用前后端分离的 B/S 架构。表现层使用 TypeScript、React、Vite、Ant Design 与 ReactFlow 构建工作台；业务服务层以 Python、FastAPI 提供 REST 与 WebSocket，通过 Celery 与 Redis 实现异步评测流水线，亦支持 Eager/Lite 进程内同步模式；数据层采用 SQLAlchemy ORM，开发环境可用 SQLite，完整环境使用 PostgreSQL，并通过 Alembic 管理迁移。

核心技术特点包括：评测领域状态机、Runner/Judge 可插拔抽象、规则与大模型混合评分、pipeline 纯函数编排、Ports & Adapters 式队列/缓存/计量适配、以及安全与多租户中间件。**由著作权人李凯昕独立完成系统架构设计、核心算法编码、领域模型定义及全流程测试验证**，形成可运行、可观测、可评分的原创软件表达，而非对单一开源组件的简单堆叠调用。

## 应用价值

本软件将 Agent 质量评估从零散脚本与人工肉眼对比，升级为可配置、可审计、可回归的工程化工作台，显著降低企业引入或迭代 Agent 时的质检成本，提升问题定位效率与评测结果可信度，适用于软件著作权登记材料展示、开源演示及企业内部 Agent 质量回归场景。
