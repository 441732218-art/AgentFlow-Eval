## 1. 软件概述

本软件是面向企业 AI 应用场景的 **Agent（智能体）自动化评测管理平台**。用户可创建评测任务、导入业务测试用例，系统自动调度 Agent 执行，记录完整推理与工具调用轨迹，并结合规则指标与大模型评审（LLM-as-Judge）生成多维评分与结构化报告。

V2.0.0 在端到端评测闭环基础上，进一步形成「用例管理 → 异步/轻量执行 → 轨迹可视化 → 混合评分 → 对比实验 → 报告与审计 → 计费与插件扩展」的完整产品表达，具有独立的数据处理流程、权限模型与界面交互设计。

## 2. 独创性要点（相对通用框架的差异化）

### 2.1 评测领域数据模型与状态机

- 自定义 **Task / TestSuite / Trace / MetricScore / Experiment / AuditLog** 等领域模型及任务状态迁移（创建、排队、运行、完成、失败、取消、归档）。
- Trace 中结构化保存 ReAct 步骤、token、耗时及版本元数据（agent/prompt/model/tool version），支撑评测可追溯。

### 2.2 Agent 执行与工具沙箱一体化

- 抽象 `BaseAgentRunner`，实现 OpenAI 兼容 **ReAct 多轮执行器**，以及 **HttpAgentRunner** 对接用户自有 Agent HTTP 服务。
- 内置 **工具沙箱**（超时控制、工具列表探测），将工具调用结果写回轨迹，而非仅保存最终自然语言回复。

### 2.3 混合评分引擎（规则 + LLM-as-Judge + 人工复核）

- `metrics` 模块实现无 LLM 的轻量规则分（如工具集合准确率、答案抽取）。
- `LLMJudge` 在规则分基础上调用大模型进行多维语义评分并生成扣分说明。
- 支持 **人工审核覆盖评分**，形成机评与人评结合的质检流程。

### 2.4 异步与轻量双部署评测流水线

- 基于 Celery + Redis 的全流程任务：逐条执行用例、落库轨迹、自动评判、更新任务状态。
- 支持 Eager / Lite 进程内同步模式便于单机演示，与生产异步模式共用业务代码路径。
- 抽出可单测的 `pipeline` 纯函数层，保证编排逻辑与框架解耦。

### 2.5 对比实验与 A/B 评测（V2）

- **Experiment** 多变体跑分与横向对比。
- **A/B 测试框架**：粘性分流、exposure/conversion/metric、统计检验与样本量估算。

### 2.6 企业级横切能力（V2）

- RBAC 五角色权限矩阵、API Key 鉴权、租户隔离、审计日志、限流与安全响应头。
- 可观测性：健康探针、Prometheus 指标、慢任务、业务 KPI。
- 插件系统与可选 SaaS 计费骨架（配额、订阅、结账 mock）。

### 2.7 前后端一体化工作台

- React 工作台：总览、任务、Trace DAG、报告、计费、插件市场等页面。
- 实时活动流（WebSocket）、主题与权限动态菜单，形成可演示的完整产品面。

## 3. 原创表达说明

本软件的原创性主要体现在：

1. **领域模型与状态机设计**：将 Agent 评测场景抽象为可持久化、可审计的业务对象，而非临时脚本集合。  
2. **执行—轨迹—评判一体化流水线**：同一套领域对象贯通执行器、轨迹存储与混合评分。  
3. **可插拔 Runner / Judge / 插件扩展点**：在保持主流程稳定的前提下支持 HTTP Agent、插件市场等扩展。  
4. **双部署剖面**：Lite 与 Private/SaaS 共用核心业务代码，降低演示与生产之间的分叉成本。  
5. **配套工作台与文档表达**：界面信息架构、报告聚合与操作手册等构成独立软件表达。

上述内容均由著作权人 **李凯昕** 独立完成架构设计、核心编码、领域模型定义与全流程验证，并非对单一开源组件的简单堆叠或界面皮肤替换。
