# 跨端打包与安装交付说明

| 属性 | 内容 |
|------|------|
| **产品** | AgentFlow Intelligence v1.0 |
| **目标** | 电脑 + 手机可安装；Windows / macOS / Linux 可运行；自适应 UI |
| **关联** | `scripts/build-release.ps1` · `desktop/` · `frontend` PWA · Docker 离线包 |

---

## 1. 交付形态一览

| 形态 | 适用端 | 三大桌面系统 | 手机 | 自适应 UI |
|------|--------|--------------|------|-----------|
| **PWA（推荐统一壳）** | 浏览器安装到主屏幕/开始菜单 | ✅ Win / macOS / Linux | ✅ Android / iOS* | ✅ |
| **Electron 桌面安装包** | 原生窗口应用 | ✅（各 OS 上构建对应包） | ❌ | ✅ |
| **Docker Private 服务端** | 服务器 / 内网 | ✅ | 浏览器访问 | ✅ |
| **离线 Docker 包** | 断网机房 | ✅ | 浏览器访问 | ✅ |

\* iOS 使用 Safari「添加到主屏幕」，无 Chrome 式 `beforeinstallprompt`。

**架构原则**：UI 一次构建，多端复用；业务 API 仍由后端提供。手机/电脑装的是 **客户端壳 + 自适应前端**，不是把 Postgres 装进手机。

```
┌──────────────────────────────────────────────────────────┐
│  Phone (PWA)  │  PC Browser (PWA)  │  Electron Desktop   │
│         自适应 SPA (React + Ant Design + responsive)       │
└───────────────────────────┬──────────────────────────────┘
                            │ HTTPS / LAN / localhost
┌───────────────────────────▼──────────────────────────────┐
│  Backend API  (Docker private / lite / SaaS)             │
│  Postgres · Redis · Celery · FastAPI                     │
└──────────────────────────────────────────────────────────┘
```

## 2. 自适应 UI（已落地）

| 断点 | 行为 |
|------|------|
| **≥992px** | 固定侧栏 + 顶栏搜索 |

## 3. 一键打包

在仓库根目录：

```powershell
# Web/PWA + 当前系统的桌面安装包
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1

# 仅 Web/PWA（手机+电脑浏览器安装）
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1 -Targets web

# Web + 桌面 + Docker 离线包
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1 -Targets web,desktop,offline

# 已 npm install 过可跳过依赖安装
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1 -SkipInstall
```

产物目录：`artifacts/release/agentflow-<时间戳>/`

| 子目录 | 内容 |
|--------|------|
| `web/` | 静态站点 + Service Worker（PWA） |
| `desktop/` | `.exe` / `.dmg` / `.AppImage` 等（取决于构建机 OS） |
| `offline/` | Docker 镜像 tar + compose（需本机已有镜像） |
| `manifest.json` | 版本与安装指引元数据 |

## 4. 安装指引

### 4.1 手机（Android）

1. 将 `web/` 部署到 **HTTPS** 域名（或受信任内网证书），或开发期用 `vite preview --host` 同一 Wi‑Fi 访问。
2. Chrome 打开站点 → 菜单 **「安装应用」** / **「添加到主屏幕」**。
3. 桌面出现 AgentFlow 图标；全屏运行，底部 Tab 导航。

### 4.2 手机（iOS）

1. Safari 打开同一站点。
2. 分享 → **添加到主屏幕**。
3. 从主屏幕打开（standalone）。

### 4.3 电脑浏览器（Win / macOS / Linux）

1. Chrome / Edge / Chromium 打开站点。
2. 地址栏安装图标，或应用内「安装 AgentFlow」横幅。
3. 出现独立窗口/开始菜单快捷方式。

### 4.4 桌面原生包

| 系统 | 文件 | 操作 |
|------|------|------|
| Windows | `AgentFlow-*-win-*.exe`（NSIS）或 `*-portable.exe` | 安装或免安装运行 |
| macOS | `*.dmg` | 拖入 Applications（需在 macOS 上构建） |
| Linux | `*.AppImage` / `*.deb` | 授权执行或 dpkg 安装 |

首次启动默认连接 `http://127.0.0.1:8000`。可设环境变量：

```text
AGENTFLOW_API_URL=https://api.example.com
```

### 4.5 服务端（内网完整能力）

```powershell
powershell -ExecutionPolicy Bypass -File scripts\docker-up.ps1
# 或离线包：见 docs/离线交付包说明.md
```

客户端用浏览器或 PWA/Electron 指向该 API。

## 5. 手动分步构建

### 5.1 仅前端 PWA

```powershell
cd frontend
npm ci
npm run build:web
# 产物 frontend/dist — 可丢 Nginx / CDN / Docker frontend 镜像
```

### 5.2 Electron

```powershell
cd frontend
npm run build:electron   # base: ./

cd ..\desktop
npm ci
npm run dist:win         # 或 dist:mac / dist:linux
# 产物 desktop/release/
```

### 5.3 后端版本

健康检查 `version` 字段为 **1.0.0**（`APP_VERSION`）。

## 6. 部署前端静态站（供手机安装）

Nginx 示例（与 API 同域最利于 PWA）：

```nginx
server {
  listen 443 ssl;
  server_name agentflow.example.com;
  root /var/www/agentflow/web;
  index index.html;

  location /api/ {
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
  }
  location /health {
    proxy_pass http://127.0.0.1:8000/health;
  }
  location / {
    try_files $uri $uri/ /index.html;
  }
}
```

PWA 要求：**安全上下文**（HTTPS 或 localhost）。纯 HTTP 局域网 IP 在部分浏览器无法触发安装。
