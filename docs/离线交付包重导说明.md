# 离线交付包重导说明（Docker 恢复后）

> 在 **Docker Desktop / 引擎已就绪** 且能 `docker info` 成功后执行。  
> 产物默认写入 `artifacts/delivery/`（已被 `.gitignore`，不进仓库）。

## 1. 前置检查

```powershell
docker info
# 应能看到 Server Version，而不是 pipe 连接错误
```

## 2. 推荐：统一发布脚本（含 offline）

在仓库根目录：

```powershell
# 仅离线包（会 build 镜像并 export tar）
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1 -Targets offline -SkipInstall

# 或 Web + Desktop + Offline 全量
powershell -ExecutionPolicy Bypass -File scripts\build-release.ps1 -Targets web,desktop,offline
```

## 3. 专用导出脚本

```powershell
powershell -ExecutionPolicy Bypass -File scripts\export-offline-package.ps1
```

常见输出：

```text
artifacts/delivery/agentflow-offline-YYYYMMDD-HHMMSS/
  images/agentflow-images.tar
  config/          # compose + env example
  scripts/         # 部署脚本
  manifest.json
  README.md
```

## 4. 目标机导入（客户侧）

```powershell
docker load -i images\agentflow-images.tar
# 再按包内 README 复制 env、compose up
```

## 5. 与 v1.0.0 对齐建议

| 项 | 建议 |
|----|------|
| 源码 / 配置 | 与 tag `v1.0.0` 或当前 `main` 一致后导出 |
| 镜像 tag | 本地 `:local` 或与 GHCR `1.0.0` 对齐后 re-tag |
| 勿打包 | 任何真实 `.env`、API Key、客户数据 |

## 6. 故障排查

| 现象 | 处理 |
|------|------|
| `dockerDesktopLinuxEngine` pipe 不存在 | 启动 Docker Desktop，等待引擎 Ready |
| 构建失败 | 先 `docker compose -f backend/docker-compose.yml build` 看日志 |
| tar 过大 | 正常（含 postgres/redis 基础镜像）；可只导出 app 镜像后客户自行 pull 基础镜像 |
