## 1. 约定（必须遵守）

| 场景 | 前端 `VITE_API_BASE_URL` | 浏览器实际请求 | 后端如何接到 |
|------|--------------------------|----------------|--------------|
| **推荐 · Docker 整体** | `/api/v1` | `http://host/api/v1/...` | nginx → `http://backend:8000` |
| **推荐 · 本地 Vite 开发** | `/api/v1` | `http://127.0.0.1:5173/api/v1/...` | Vite proxy → `http://127.0.0.1:8000` |
| 直连调试 | `http://127.0.0.1:8000/api/v1` | 跨域 | 依赖 CORS |

默认与 Docker 构建参数均为 **`/api/v1`**（同源相对路径）。

---

## 2. 本地开发（后端已是 Docker 时）

1. 后端容器：`agentflow-backend` 监听 `8000`  
2. 前端：

```powershell
cd D:\AgentFlow-Eval\frontend
# .env.local 已含 VITE_API_BASE_URL=/api/v1
npm run dev
```

3. 打开 **http://127.0.0.1:5173**  
   - 页面内 API 走 `/api/v1/*`  
   - Vite 自动代理到 `http://127.0.0.1:8000`

验证：

```powershell
# 代理是否通
Invoke-RestMethod http://127.0.0.1:5173/api/v1/me
Invoke-RestMethod http://127.0.0.1:8000/api/v1/me
```

## 3. Docker 整体打包运行

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-docker-stack.ps1
# 可选种子：
powershell -ExecutionPolicy Bypass -File scripts\start-docker-stack.ps1 -Seed
# 已构建过：
powershell -ExecutionPolicy Bypass -File scripts\start-docker-stack.ps1 -SkipBuild -Seed
```

| 地址 | 用途 |
|------|------|
| http://127.0.0.1/ | **统一入口**（前端 + `/api` 反代） |
| http://127.0.0.1/api/v1/me | 经 nginx 的 API |
| http://127.0.0.1:8000/health/ready | 直连 API |
| http://127.0.0.1/health/ready | 经 nginx 的 ready |

组件：`postgres` · `redis` · `backend` · `celery-worker` · `frontend(nginx)`

## 4. 配置文件索引

| 文件 | 作用 |
|------|------|
| `frontend/.env.local` | 本地 Vite 同源 `/api/v1`（不提交密钥） |
| `frontend/.env.example` | 模板说明 |
| `frontend/vite.config.ts` | dev 代理 `/api`、`/health` |
| `frontend/nginx.conf` | 生产反代 `/api`、`/health/ready`、`/metrics` |
| `frontend/Dockerfile` | 构建时注入 `VITE_API_BASE_URL=/api/v1` |
| `backend/docker-compose.yml` | 全栈服务与 frontend 依赖 backend healthy |
| `scripts/start-docker-stack.ps1` | 一键启动 |

---

## 5. 排障

| 现象 | 处理 |
|------|------|
| 前端空白 / 网络错误 | 确认 backend `:8000` healthy；检查 `.env.local` 为 `/api/v1` 后 **重启 Vite** |
| CORS 报错 | 改回同源 `/api/v1`，不要用跨域绝对 URL；或把前端源加入 `CORS_ORIGINS` |
| Docker UI 能开但接口 502 | `docker logs agentflow-frontend` / `agentflow-backend`；确认同网络 `backend_agentflow-net` |
| 改 API 地址不生效 | Vite/前端需 **重新 build**（`VITE_*` 构建期注入） |

## 6. 交付默认（生产向）

| 项 | 默认 | 说明 |
|----|------|------|
| `AUTH_ENABLED` | `true` | 需 `X-API-Key`；前端设置页填写 |
| Postgres/Redis 端口 | `127.0.0.1` 绑定 | `docker-compose.prod.yml` 则完全不映射宿主机 |
| `/docs` | 关闭 | `ENV=prod` |
| 离线包 | `scripts/export-offline-package.ps1` | `docker save` + compose + 说明 |

```powershell
# 干净重建 + 启动
powershell -ExecutionPolicy Bypass -File scripts\start-docker-stack.ps1
# 校验（自动读 backend/.env.docker 的 API_KEYS）
powershell -ExecutionPolicy Bypass -File scripts\post-deploy-verify.ps1
# 导出离线交付目录
powershell -ExecutionPolicy Bypass -File scripts\export-offline-package.ps1
```
