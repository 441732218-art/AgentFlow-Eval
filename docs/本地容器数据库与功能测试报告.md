### 3.2 未通过（❌）与原因

| 项 | 现象 | 原因与处理 |
|----|------|------------|
| `GET /ab/experiments` | 404 | **路径错误**：列表应为 **`GET /api/v1/ab`**；`/ab/{id}` 把 `experiments` 当成了 id |
| `POST .../test-suites` | 422 | Body 须为 **JSON 数组** `[{...}]`，不是 `{"items":[...]}` |
| `POST .../execute` | 超时 | Eager 模式下评测在请求线程跑 **OpenAI**，外网/密钥慢导致阻塞 API |
| `GET /reports/{id}` | 超时 | 受 execute 阻塞连带 |
| `GET /diagnosis/task/{id}`（新任务） | 超时 | 同上 |

**说明：** Demo 任务的诊断列表在阻塞前已 **200**；完整 `execute` 需有效 `OPENAI_API_KEY` + 网络，或改用 **Celery Worker 容器** 异步（`CELERY_TASK_ALWAYS_EAGER=false`）。

## 5. 常用命令备忘

```powershell
# 启动库
cd D:\AgentFlow-Eval\backend
docker compose --env-file .env.docker up -d postgres redis

# 迁移 + 种子（本机，指向 127.0.0.1:5432）
$env:DATABASE_URL="postgresql+asyncpg://agentflow:<密码>@127.0.0.1:5432/agentflow_eval"
$env:PYTHONPATH=(Get-Location).Path
python -m alembic upgrade head
python -m app.core.seed --force

# 启动 API（同上 DATABASE_URL；开发可用 eager）
$env:CELERY_TASK_ALWAYS_EAGER="true"
$env:DEPLOY_PROFILE="lite"
$env:ENV="dev"
$env:AUTH_ENABLED="false"
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

# 功能冒烟
python scripts_local_api_test.py
```
