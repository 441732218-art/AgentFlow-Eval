# AgentFlow-Eval 用户操作手册

## 1. 安装部署指南

### 1.1 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Python | 3.10 | 3.12 |
| Node.js | 18 | 22 |
| npm | 9 | 11 |
| Redis（可选） | 7 | 7 |

### 1.2 快速启动

#### 后端启动

```bash
# 1. 克隆项目后进入后端目录
cd AgentFlow-Eval/backend

# 2. 创建并激活虚拟环境
python -m venv .venv
# Windows: .venv\Scripts\activate
# Mac/Linux: source .venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置 OPENAI_API_KEY=sk-your-key-here

# 5. 启动服务
uvicorn app.main:app --reload --port 8000
```

启动后访问 http://localhost:8000/docs 查看自动生成的 API 文档（Swagger UI）。

#### 前端启动

```bash
# 1. 进入前端目录
cd AgentFlow-Eval/frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

启动后访问 http://localhost:5173 即可看到评测工作台界面。

> **说明**: 前端 Vite 已配置代理，`/api` 开头的请求会自动转发到后端 http://localhost:8000。

#### 本地评测模式（无需 Redis）

若本地不启动 Redis 和 Celery Worker，在 `backend/.env` 中设置：

```bash
CELERY_TASK_ALWAYS_EAGER=true
```

这样点击「Execute」时，评测任务会在 API 进程内同步执行，无需额外基础设施。生产环境请保持 `false` 并使用 Redis + Celery Worker。

#### 启动 Docker 基础设施（可选）

如果有 Redis 或 PostgreSQL 需求：

```bash
cd backend
docker-compose up -d
```

### 1.3 验证安装

访问 http://localhost:8000/health，应返回类似：

```json
{
  "status": "healthy",
  "app": "AgentFlow-Eval",
  "version": "0.1.0",
  "services": {
    "database": "ok",
    "redis": "ok"
  }
}
```

> 未启动 Redis 时 `redis` 字段为 `unavailable`，整体状态为 `degraded`，不影响本地 eager 模式运行。

## 2. 创建评测任务

### 2.1 填写任务信息

在左侧「Task Configuration」区域填写：

1. **Task Name**（必填）：输入任务名称，如 "客户服务 Agent 评测"
2. **Description**（可选）：输入描述信息
3. **Agent Model**：选择 Agent 使用的模型（GPT-4o / GPT-4o-mini / GPT-4 Turbo）
4. **Temperature**：模型温度参数（0-2），默认 0
5. **Max Tokens**：最大 Token 数，默认 4096

### 2.2 上传测试用例

在右侧「Upload Test Cases」区域上传 CSV 或 JSON 文件：

#### CSV 格式

```csv
query,expected_output,expected_tools
北京天气怎么样？,北京今天天气晴朗,get_weather
计算 15*37,555,calculator
用 Python 读取文件内容,使用 open() 函数,code_interpreter
```

**CSV 列说明**：
- `query`（必填）：用户输入的测试指令
- `expected_output`（可选）：期望的 Agent 输出
- `expected_tools`（可选）：期望 Agent 调用的工具名称（多个用逗号分隔）

#### JSON 格式

```json
[
  {
    "query": "北京天气怎么样？",
    "expected_output": "北京今天天气晴朗",
    "expected_tools": ["get_weather"]
  },
  {
    "query": "计算 15*37",
    "expected_output": "555",
    "expected_tools": ["calculator"]
  }
]
```

支持字段名：`query` 或 `user_query`。

### 2.3 创建任务

1. 拖拽或点击上传 CSV/JSON 文件
2. 上传成功后，右侧会显示解析结果预览表格
3. 确认测试用例正确后，点击左下角的「Create Task」按钮
4. 系统会先创建任务，再批量导入测试用例
5. 创建成功后自动跳转到任务详情页

> **提示**: 上传后可通过「Clear」按钮清空已解析的测试用例。

## 3. 查看评测结果

### 3.1 任务列表页

通过左侧导航栏点击「Tasks」进入任务列表页。

列表显示所有评测任务，每行包含：
- 任务名称
- 当前状态（pending / running / completed / failed）
- 测试用例数量
- 创建时间
- 操作按钮（查看详情 / 执行评测 / 删除任务）

### 3.2 评测详情页

点击任务名称或「详情」按钮进入评测详情页，页面包含：

**顶部信息栏**：
- 任务名称、状态标签、测试用例数
- 综合评分（如果有评分结果）
- 总 Token 消耗和平均耗时
- 「Run」按钮触发评测执行，「Refresh」按钮刷新

**评分概览**（评测完成后显示）：
- 三个维度独立评分卡片
- 工具调用准确率 / 答案准确性 / 推理链路合理性

**测试用例详情**：
- 每个测试用例的执行状态
- Token 消耗、耗时、评分
- 「Judge」按钮触发评分，「Trace」按钮查看执行轨迹

### 3.3 链路图交互

点击「Trace」按钮后，下方显示 ReactFlow 执行链路 DAG 图：

- **节点颜色**: Thought(蓝) → Action(金) → Observation(绿) → Final Answer(紫)
- **节点内容**: 步骤类型标签 + 序号 + 工具名 + 内容摘要
- **边**: 带箭头和 Step N 标签的动画曲线
- **交互**:
  - 点击节点 → 选中高亮 → 下方显示详情面板
  - 鼠标悬停 → Tooltip 显示完整内容
  - 滚轮缩放 → 0.4x ~ 2x
  - Ctrl+拖拽 → 平移视图
  - 右上角 MiniMap → 概览导航
  - 左下角 Controls → 缩放 / 适应视图

### 3.4 评分卡片

右侧「Scoring Results」卡片显示：
- 仪表盘总分（环形进度条）
- 各维度分数字段（tool_accuracy/answer_correctness/reasoning_coherence）
- 评分说明（扣分/加分理由）

### 3.5 步骤日志

「Step Log」面板以时间线形式展示每个 ReAct 步骤的详细内容：
- 步骤类型标签（颜色编码与链路图一致）
- 工具名称和输入参数
- 完整执行内容（超过 200 字符自动截断）

## 4. 解读评分报告

### 4.1 三个评分维度

#### 工具调用准确率（满分 40 分）

评估 Agent 是否正确调用了预期工具。评分规则：

| 扣分项 | 扣分 |
|--------|------|
| 缺少一个必要工具 | -10 分 |
| 使用了预期外的工具 | -10 分 |
| 工具参数明显错误 | -5 分/次 |

#### 答案准确性（满分 40 分）

评估 Agent 的最终回答是否包含预期输出的核心信息。评分规则：

| 扣分项 | 扣分 |
|--------|------|
| 遗漏核心信息 | -15 分 |
| 信息不准确 | -10 分 |
| 回答过于笼统 | -5 分 |

#### 推理链路合理性（满分 20 分）

评估 Agent 的思考过程是否合理。评分规则：

| 扣分项 | 扣分 |
|--------|------|
| 明显的重复思考 | -8 分 |
| 逻辑跳跃或错误推理 | -6 分 |
| 步骤冗余 | -4 分 |

### 4.2 总分计算

```
总分 = tool_accuracy(0-40) + answer_correctness(0-40) + reasoning_coherence(0-20)
     = 0-100 分
```

**评分说明**：
- 无 LLM API Key 时：纯规则评分，返回 `mode: "rule_only"`
- 有 LLM API Key 时：规则 + LLM 混合评分，返回 `mode: "hybrid"`
- LLM 评分失败时：自动回退到规则评分

### 4.3 报告导出

在评测详情页顶部可查看完整的聚合报告，包含：
- 测试用例总数、成功/失败数
- 各维度平均分
- Token 消耗和耗时统计
- 每个测试用例的详细执行结果

## 5. 常见问题 FAQ

### Q1: 前端启动后页面空白怎么办？

1. 确认已执行 `npm install` 安装依赖
2. 检查控制台是否有报错信息
3. 确认 Node.js 版本 ≥ 18
4. 尝试清除 `node_modules` 后重新安装

### Q2: 后端启动报错 ModuleNotFoundError？

```bash
# 确认已安装所有依赖
pip install -r requirements.txt

# 确认在正确的目录下执行
cd AgentFlow-Eval/backend
```

### Q3: 评测任务执行失败？

1. 确认已设置有效的 `OPENAI_API_KEY`
2. 检查测试用例格式是否正确
3. 查看后端日志 `logs/agentflow_eval.log` 获取详细错误信息
4. 确认 Redis 服务已启动（如果使用 Celery 模式）

### Q4: 没有 OpenAI API Key 能否使用？

可以。系统会自动检测 API Key：
- 有 Key：启用混合评分模式（规则 + LLM）
- 无 Key：启用纯规则评分模式（工具准确率 + 词汇重叠 + 推理启发式）

Agent 执行需要 API Key；评分引擎可以在无 Key 模式下运行。

### Q5: 上传 CSV/JSON 时提示格式错误？

**CSV 格式要求**：
- 第一行必须为列头：`query,expected_output,expected_tools`
- `query` 列为必填
- 如果字段值包含逗号，请用双引号包裹：`"value,with,commas"`

**JSON 格式要求**：
- 必须是数组：`[...]`
- 每个元素必须包含 `"query"` 或 `"user_query"` 字段

### Q6: 如何重置数据库？

删除项目根目录下的 `agentflow_eval.db` 文件，重启后端服务即可自动重建。

### Q7: 支持哪些语言？

目前 Agent 提示词和 LLM 评分提示词支持中文和英文。前端界面使用英文标签，但包含中文字符注释。
