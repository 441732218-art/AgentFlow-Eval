## 生成

前提：本机已干净构建镜像（`agentflow-backend:local` / `agentflow-frontend:local`）。

```powershell
# 推荐：先重建并校验
cd backend
docker compose --env-file .env.docker build --no-cache backend celery-worker frontend
docker compose --env-file .env.docker up -d
powershell -ExecutionPolicy Bypass -File ..\scripts\post-deploy-verify.ps1

# 导出
powershell -ExecutionPolicy Bypass -File scripts\export-offline-package.ps1
```

输出目录示例：

`artifacts/delivery/agentflow-offline-YYYYMMDD-HHMMSS/`

| 路径 | 内容 |
|------|------|
| `images/agentflow-images.tar` | 镜像离线包（backend/frontend/postgres/redis） |
| `config/docker-compose.yml` | 本地/私有化 compose（DB/Redis 仅绑 127.0.0.1） |
| `config/docker-compose.prod.yml` | 生产 compose（DB/Redis 不映射宿主机端口） |
| `config/.env.docker.example` | 环境变量模板（**无密钥**） |
| `scripts/*` | 启动 / 校验脚本副本 |
| `README.md` | 英文装机说明 |
| `manifest.json` | 导出元数据 |

## 客户机装机

见包内 `README.md`。要点：

1. `docker load -i images\agentflow-images.tar`
2. 复制并编辑 `.env.docker`（必开 `AUTH_ENABLED=true` + `API_KEYS`）
3. `docker compose --env-file .env.docker up -d`
4. 浏览器打开 `http://host/`，在「设置」填写 API Key
5. `post-deploy-verify.ps1 -ApiKey ...` 验收

## 交付默认（与运行栈一致）

| 项 | 值 |
|----|-----|
| AUTH | 默认开启 |
| Postgres/Redis 端口 | compose 绑 `127.0.0.1`；prod 不映射 |
| Swagger `/docs` | prod 关闭 |
| 镜像标签 | `agentflow-*:local`（可改 tag 后改 `BACKEND_IMAGE`/`FRONTEND_IMAGE`） |

## 注意

- **不要**把含真实密钥的 `backend/.env.docker` 打进交付包。
- `artifacts/` 已 gitignore，离线 tar 仅本机/U 盘分发。
